## 3. 什么是因特网

### 3.1. 具体构成描述

因特网可以从具体构成的角度来理解——它是由大量设备和连接组成的全球性网络。

接入因特网的设备被称为端系统（主机），比如你的手机、笔记本电脑、服务器等。端系统通过通信链路相互连接，这些链路由不同的物理媒体构成，包括光纤、铜线、无线电波等，传输速率用 bit/s 来度量。数据在传输时被分割成小块，称为分组，分组通过分组交换机（最常见的两种是路由器和链路层交换机）逐跳转发。从发送端到接收端，分组经过的一系列通信链路和交换机被称为路径。

端系统并非直接互联，而是通过因特网服务提供商（ISP）接入因特网。ISP 之间也相互连接，形成层次化的网络结构。所有这些设备和连接都遵循一套共同的规则来通信，这套规则就是协议，例如 TCP、IP、HTTP 等。

### 3.2. 服务描述

从应用程序的角度看，因特网不仅仅是硬件的集合，它更是一个为分布式应用提供服务的基础设施。

分布式应用程序运行在端系统上，利用因特网提供的通信服务相互交换信息，例如 Web 浏览、视频流媒体、电子邮件等。因特网为应用程序提供了两类基本的传输服务：面向连接的可靠服务（TCP），保证数据按序、无差错地到达；以及无连接的不可靠服务（UDP），不保证交付但更轻量高效。应用程序通过套接字接口（Socket API）向因特网交付数据，这是端系统与网络之间的约定。

### 3.3. 什么是协议

协议定义了在两个或多个通信实体之间交换报文的格式和顺序，以及报文发送和接收时所采取的动作。可以类比人类社交中的礼仪规范——它规定了在什么场景下应该说什么话、做什么事。

一个网络协议包含三个关键要素：语法定义报文的格式，即字段如何排列、各占多少比特；语义定义每个字段的含义以及接收方应如何响应；时序定义报文交换的顺序和速率。这三者缺一不可，共同确保通信双方能够正确理解彼此。

## 4. 网络边缘

### 4.1. 接入网

接入网是指将端系统物理连接到其边缘路由器的网络，它是用户通往因特网的第一段路。

在家庭场景中，最常见的接入方式有三种：数字用户线（DSL）利用已有的电话线传输数据，通过频分复用让数据和语音同时在线路上传输，数据从家庭到达本地中心局后进入 ISP 的网络；电缆因特网接入（HFC）借助有线电视的同轴电缆提供网络接入，带宽在多个用户之间共享；光纤到户（FTTH）直接将光纤铺设到家庭，提供极高的传输速率，是未来家庭接入的发展方向。

在企业场景中，以太网是最流行的接入技术，使用双绞线将端系统连接到交换机；WiFi则基于 IEEE 802.11 标准提供无线局域网接入，让设备无需物理线缆即可联网。

在广域范围，4G/5G 蜂窝网络通过基站为移动设备提供无线接入，覆盖范围远超 WiFi，但带宽通常更低。

### 4.2. 物理媒介

物理媒介是比特传输的载体，分为导引型媒体和非导引型媒体两大类。

导引型媒体中，信号沿固体传播。双绞铜线是最便宜、最常用的传输媒体，由两根相互绝缘的铜线绞合而成，广泛用于以太网和电话网络；同轴电缆由两个同心铜导体组成，常用于有线电视和 HFC 接入网；光纤利用光脉冲传输数据，速率极高、抗电磁干扰、衰减极低，适合长距离和高带宽场景。

非导引型媒体中，信号在空气或外层空间中自由传播，包括 WiFi 使用的无线局域网、4G/5G 使用的蜂窝移动通信，以及覆盖偏远地区的卫星通信。

## 5. 网络核心

### 5.1. 分组交换

在因特网的核心，数据以分组（packet）为单位进行传输。源端系统将长报文分割为较小的数据块，每个分组独立通过通信链路和分组交换机传送到目的地。

分组交换的一个关键机制是存储转发传输——交换机必须接收到一个分组的全部比特后，才能开始向出链路转发该分组的第一个比特。这意味着如果一个分组由 $L$ 比特组成，链路传输速率为 $R$ bit/s，那么该分组引入的存储转发时延为 $L/R$。如果路径上有 $N$ 条链路，端到端的总时延大约为 $NL/R$。

每台交换机为每条输出链路维护一个输出缓存（队列）。当分组到达的速率超过输出链路的转发速率时，分组就需要在队列中排队等待，产生排队时延。如果缓存空间被占满，后续到达的分组就会被丢弃，即发生丢包。

每台路由器维护一张转发表，将分组的目的 IP 地址映射到对应的输出链路。转发表由路由选择协议自动计算和维护，确保分组能沿着正确的路径逐跳到达目的地。

### 5.2. 电路交换

与分组交换不同，电路交换在通信开始前需要沿路径预留资源（如链路带宽），建立一条专用的端到端连接。一旦连接建立，发送方可以以恒定速率传输数据，不受其他连接的干扰。

电路交换网络使用两种多路复用技术来共享链路：频分复用（FDM）将链路的频谱划分为多个频段，每条连接专用一个频段；时分复用（TDM）将时间划分为固定长度的帧，每条连接在每帧中占用固定的时隙。

两者的根本区别在于资源分配方式。电路交换预留资源，保证了稳定的性能，但在连接空闲时带宽被浪费，资源利用率低。分组交换不预留资源，多个连接共享带宽，实现简单且资源利用率高，但可能面临排队时延和丢包的问题。正因如此，分组交换成为了因特网的选择。

### 5.3. 网络的网络

因特网不是由单个网络构成的，而是由众多 ISP 组成的层次化结构，可以把它想象成一个「网络的网络」。

在最顶层是数量很少的第一层 ISP，它们相互之间直接对等连接，覆盖国际区域，构成因特网的骨干。第二层是区域 ISP，它们连接第一层 ISP 和更下层的本地 ISP。最底层是接入 ISP，直接面向端用户提供接入服务。

ISP 之间通过存在点（POP）相互连接——POP 是 ISP 网络中的一台或多台路由器组群，其他 ISP 可以在此与之建立连接。多个 ISP 还可以在互联网交换点（IXP）汇聚并相互对等连接，降低互联成本。

此外，大型内容提供商如 Google、Netflix 等自建了专用网络，直接与较低层 ISP 连接，绕过第一层 ISP，既降低了成本又提升了服务质量。这使得因特网的整体结构变得更加复杂，形成了一个多层次、多路径的生态系统。
