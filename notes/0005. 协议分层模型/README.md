## 3. 分层的体系结构

### 3.1. 应用层

应用层是协议栈的最顶层，直接为用户的应用程序提供网络服务。因特网中常见的应用层协议包括 HTTP（用于 Web 浏览）、SMTP（用于电子邮件传输）、FTP（用于文件传输）和 DNS（用于域名解析）。应用层协议分布在端系统上，定义了应用进程之间交换报文的格式和规则。

应用层的消息分组被称为报文（message），它是应用层协议数据的基本单位。

### 3.2. 运输层

运输层负责在两个端系统之间提供进程到进程的通信服务。因特网中有两种主要的运输层协议：TCP 提供面向连接的可靠数据传输，包括流量控制、拥塞控制和数据完整性保障；UDP 则提供无连接的轻量级服务，不保证可靠性但延迟更低。

运输层接收应用层的报文，将其分割为较小的单元称为报文段（segment），并为每个报文段添加运输层首部信息。

### 3.3. 网络层

网络层负责将数据报（datagram）从源主机传送到目的主机。它接收运输层的报文段，封装成数据报，并通过路由选择算法确定数据报从源到目的地的传输路径。

因特网的网络层协议是 IP 协议，它定义了数据报的格式以及地址格式（IPv4 和 IPv6）。IP 协议提供的是尽力而为（best-effort）的交付服务，不保证数据报一定能到达、不保证顺序、不保证不重复。

### 3.4. 链路层

链路层负责将数据报从一个节点（主机或路由器）通过单一通信链路传送到相邻节点。网络层的数据报在传输过程中可能经过多条不同的链路，每条链路可能使用不同的链路层协议，如以太网、WiFi、PPP 等。

链路层的消息分组被称为帧（frame），它将网络层的数据报封装并添加链路层首部和尾部信息，包括源和目的 MAC 地址、差错检测码等。

### 3.5. 物理层

物理层是协议栈的最底层，负责将帧中的一个个比特从一个节点传送到相邻节点。它定义了传输媒体上的电气、时序和接口特性，比如电压高低代表 0 还是 1、连接器的针脚数量和排列方式等。

物理层协议与具体的传输媒体密切相关，不同媒体（双绞线、光纤、无线电波）对应不同的物理层规范。

## 4. 封装

封装是分层协议架构中的核心机制。数据从应用层向下传递到物理层的过程中，每一层都会在数据前面添加本层的首部信息（链路层还会添加尾部），形成该层的协议数据单元。

具体过程是这样的：应用层产生报文，传递给运输层；运输层添加首部后形成报文段，传递给网络层；网络层添加首部后形成数据报，传递给链路层；链路层添加首部和尾部后形成帧，最终由物理层以比特流的形式发送到链路上。

接收端的处理过程恰好相反：每一层读取并处理本层的首部信息后，将数据部分向上传递，逐步解封装，直到应用层获得原始报文。这种分层封装的设计使得每一层都可以独立工作，只需关注自己的职责，而不必了解其他层的实现细节。这也意味着替换某一层的实现（比如将有线网络换成 WiFi）不会影响上层应用的正常运行。
