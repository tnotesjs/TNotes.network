## 3. Web 和 HTTP 概述

### 3.1. Web 页面

Web 页面（也称文档）是由对象组成的。一个对象就是一个文件，比如 HTML 文件、JPEG 图片、JavaScript 脚本等，每个对象都可以通过 URL 来寻址。Web 页面通常包含一个 HTML 基本文件以及若干个引用对象。HTML 基本文件通过对象的 URL 来引用页面中的其他对象，浏览器解析 HTML 时会发现这些引用并逐一发起请求。

### 3.2. HTTP 概述

HTTP（超文本传输协议）是 Web 的应用层协议，定义了客户端与服务器之间交换报文的格式和规则。HTTP 使用 TCP 作为运输层协议，客户端发起连接后，通过 TCP 可靠地传输请求和响应报文。HTTP 是一个无状态协议，服务器不会保存关于客户端的任何历史信息，这意味着每次请求都是独立的，服务器不会记住之前发生过什么。这种设计简化了服务器的实现，使其可以高效地服务大量客户端。

## 4. 非持续连接和持续连接

### 4.1. 非持续连接

非持续连接是指每次请求/响应对都使用一个独立的 TCP 连接。在 HTTP/1.0 中默认使用非持续连接。当浏览器请求一个包含多个对象的 Web 页面时，每个对象都需要单独建立一个 TCP 连接，传输完成后立即关闭。

非持续连接的主要缺点有两个。首先是每个对象都要经历 TCP 三次握手的开销，增加了响应时间。其次是每个 TCP 连接只传输一个对象后就关闭了，连接资源没有被充分利用。假设一个页面包含 1 个 HTML 文件和 10 个图片，浏览器就需要建立 11 个 TCP 连接。

### 4.2. 持续连接

持续连接是指在同一个 TCP 连接上可以传输多个请求和响应。HTTP/1.1 默认使用持续连接。服务器在发送响应后保持 TCP 连接打开，后续的请求和响应可以继续在这个连接上传输，直到某一方主动关闭连接。

持续连接大幅减少了 TCP 连接建立的开销，降低了服务器的负担，也减少了网络中的分组数量。此外，持续连接还支持流水线机制——客户端可以在收到前一个响应之前就发送下一个请求，进一步提高了传输效率。

## 5. HTTP 报文格式

### 5.1. HTTP 请求报文

HTTP 请求报文由请求行、请求头、空行和消息体四部分组成。请求行包含三个字段：方法（如 GET、POST、PUT、DELETE）、请求的 URL 路径和 HTTP 版本号。请求头由多行「字段名: 值」组成，常见的字段包括 Host（目标主机名）、User-Agent（客户端类型）、Accept（可接受的响应格式）、Connection（连接管理方式）等。空行用于分隔请求头和消息体。GET 请求通常没有消息体，POST 请求在消息体中携带要提交给服务器的数据。

### 5.2. HTTP 响应报文

HTTP 响应报文由状态行、响应头、空行和消息体四部分组成。状态行包含 HTTP 版本号、状态码和原因短语。响应头包含服务器信息、内容类型、内容长度、日期等字段。消息体中包含服务器返回的实际内容，如 HTML 文件、图片数据或 JSON 数据等。

### 5.3. 状态码

HTTP 状态码由三位数字组成，分为五类。1xx 表示信息性响应，如 100 Continue。2xx 表示成功，最常见的是 200 OK，表示请求成功处理。3xx 表示重定向，如 301 Moved Permanently（资源永久移至新 URL）和 302 Found（临时重定向）。4xx 表示客户端错误，常见的有 400 Bad Request（请求语法错误）、403 Forbidden（服务器拒绝请求）、404 Not Found（资源不存在）。5xx 表示服务器错误，如 500 Internal Server Error（服务器内部错误）和 503 Service Unavailable（服务暂不可用）。
