## 3. Web 缓存

### 3.1. 什么是 Web 缓存

Web 缓存器（也叫代理服务器）是能够代表原始服务器来满足 HTTP 请求的网络实体。它有自己的存储空间，保存了最近请求过的对象的副本。当客户端发送请求时，请求首先被路由到 Web 缓存器，如果缓存器中已经保存了请求的对象副本，就直接将其返回给客户端，而不需要将请求转发到原始服务器。

### 3.2. 缓存的工作原理

缓存的工作流程如下。浏览器首先与 Web 缓存器建立 TCP 连接，并向它发送 HTTP 请求。缓存器收到请求后，检查本地是否保存了该对象的副本。如果命中缓存，缓存器直接将对象返回给浏览器，这称为缓存命中。如果未命中，缓存器会与原始服务器建立 TCP 连接，向其发送 HTTP 请求获取对象。原始服务器返回对象后，缓存器在本地保存一份副本，然后将对象返回给浏览器。

### 3.3. 条件 GET 请求

缓存面临的一个问题是：保存的对象副本可能已经过期。HTTP 通过条件 GET 请求来解决这个问题。条件 GET 请求在请求头中包含 If-Modified-Since 字段，其值为上次获取该对象时服务器返回的 Last-Modified 时间。服务器收到条件 GET 请求后，检查对象是否在该时间之后被修改过。如果没有修改，服务器返回 304 Not Modified 响应，不包含消息体，告诉缓存器可以继续使用缓存的副本。如果对象已被修改，服务器返回 200 OK 和新的对象内容。这种方式既保证了数据的新鲜度，又减少了不必要的数据传输。

### 3.4. 缓存的好处

Web 缓存带来了多方面的好处。首先，它显著减少了客户端的请求响应时间，因为从附近的缓存获取数据比从远处的原始服务器获取要快得多。其次，缓存大大减少了因特网上的 Web 流量，减轻了服务器和网络链路的负担。对于机构来说，部署 Web 缓存器可以降低带宽需求，节省网络成本。此外，缓存还能在原始服务器暂时不可用时提供一定的容错能力。

## 4. 内容分发网

### 4.1. 什么是 CDN

内容分发网（CDN）是分布在多个地理位置的服务器集群，它将内容（如视频、图片、网页）缓存到离用户更近的边缘节点上，使用户可以从距离最近的服务器获取内容。CDN 是对 Web 缓存思想的大规模扩展，从单一缓存器发展为全球分布的缓存网络。

### 4.2. CDN 的工作原理

CDN 的核心机制是请求重定向。当用户请求某个资源时，DNS 系统不会返回原始服务器的 IP 地址，而是返回由 CDN 管理的某个边缘服务器的地址。CDN 通过 DNS 拦截和 HTTP 重定向等技术，将用户的请求引导到最合适的边缘节点。选择节点时通常会考虑地理位置的远近、服务器的负载状况、网络链路的质量等因素。

### 4.3. CDN 的部署策略

CDN 有两种主要的部署策略。集群选择策略是在少数几个关键位置部署大型服务器集群，每个集群包含大量服务器。这种策略管理简单，但对远离集群的用户体验较差。另一种是分布式策略，在全球各地部署大量的小型节点，尽可能靠近终端用户。分布式策略能提供更低的延迟，但管理和维护成本更高。实际的 CDN 通常是两者的混合。

### 4.4. 案例——Netflix 和 YouTube

Netflix 和 YouTube 是全球最大的视频流媒体服务，它们都大量使用了 CDN 技术。Netflix 自建了 Open Connect CDN，将视频内容预加载到部署在全球 ISP 网络中的专用服务器上。用户观看视频时，数据从最近的 Open Connect 服务器流出，而非从 Netflix 的中央数据中心传输。YouTube 则使用 Google 的全球 CDN 基础设施，通过多码率自适应、内容预取等技术优化视频播放体验。这两个案例充分说明了 CDN 对大规模内容分发的重要性。
