## 3. DNS 提供的服务

### 3.1. 主机名到 IP 地址的转换

DNS 最核心的功能是将人类可读的主机名（如 www.example.com）转换为网络设备使用的 IP 地址（如 93.184.216.34）。这个过程称为域名解析。当用户在浏览器中输入一个 URL 时，浏览器首先需要通过 DNS 查询获得目标服务器的 IP 地址，然后才能建立 TCP 连接并发送 HTTP 请求。DNS 通过一个全球分布的分布式数据库系统来完成这一转换，避免了单一服务器成为性能瓶颈和单点故障。

### 3.2. DNS 的其他服务

除了主机名到 IP 地址的转换，DNS 还提供其他几种重要服务。邮件服务器别名（MX 记录）允许将邮件地址中的域名映射到实际的邮件服务器地址，使得机构可以使用与 Web 服务器不同的邮件服务器。负载均衡通过返回多个 IP 地址来实现，DNS 可以将请求分散到不同的服务器上。主机别名（CNAME 记录）允许为同一个主机设置多个域名。DNS 还可以用于服务发现，帮助客户端找到提供特定服务的服务器。

## 4. DNS 工作机理概述

### 4.1. 分布式层次数据库

DNS 采用分布式、层次化的架构，由三类 DNS 服务器组成。根 DNS 服务器位于层次结构的最顶层，全球共有 13 组根服务器（标号 A 到 M），每组由多台实际服务器通过任播技术部署在全球各地。顶级域（TLD）服务器负责管理顶级域名，如 .com、.org、.cn 等。权威 DNS 服务器负责存储特定域名的 DNS 记录，通常由域名的注册者或其 ISP 管理。

此外还有本地 DNS 服务器，它严格来说不属于 DNS 的层次结构，但扮演着重要的中介角色。当主机发出 DNS 查询请求时，请求首先被发送到本地 DNS 服务器，由它代理向 DNS 层次结构发起递归或迭代查询。

### 4.2. DNS 缓存

DNS 缓存是提高 DNS 查询效率的关键机制。当 DNS 服务器收到一个查询并成功获得答案后，会将结果缓存在本地。下次收到相同域名的查询时，可以直接从缓存中返回结果，无需再次遍历 DNS 层次结构。缓存的条目会在 TTL 指定的时间后过期。由于缓存的存在，大多数 DNS 查询实际上不需要到达根服务器或 TLD 服务器，大大减少了查询延迟和根服务器的负载。

## 5. DNS 记录和报文

### 5.1. DNS 记录

DNS 数据库中存储的每条记录称为资源记录（RR），它是一个四元组，包含 Name、Value、Type 和 TTL 四个字段。Type 决定了记录的含义：A 记录将主机名映射到 IPv4 地址；AAAA 记录映射到 IPv6 地址；CNAME 记录定义主机的别名；MX 记录指定域名的邮件服务器。例如一条 A 记录 (www.example.com, 93.184.216.34, A, 300) 表示 www.example.com 对应的 IPv4 地址是 93.184.216.34，缓存时间为 300 秒。

### 5.2. DNS 报文

DNS 报文分为查询报文和响应报文，它们使用相同的格式，由首部和四个区域组成。首部包含事务 ID、标志位（如查询/响应、权威回答、递归期望等）和各区域的记录数。问题区域包含查询的域名、类型和类别。回答区域包含响应的资源记录。权威区域包含其他权威服务器的记录。附加区域包含可能有用的附加信息。DNS 通常使用 UDP 协议传输，端口号为 53，报文大小限制为 512 字节（超过时使用 TCP）。

### 5.3. 在 DNS 数据库中插入记录

要让一个新域名在互联网上可访问，需要在 DNS 系统中注册相应的记录。首先需要向域名注册商注册一个域名，然后指定该域名的权威 DNS 服务器。在权威 DNS 服务器上添加 A 记录、MX 记录等资源记录。最后需要在 TLD 服务器上添加 NS 记录和胶水记录，将域名指向你的权威 DNS 服务器。完成这些步骤后，全球的 DNS 查询就能找到你的域名对应的 IP 地址了。
