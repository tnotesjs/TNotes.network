## 3. 电子邮件系统的组成

### 3.1. 用户代理

用户代理是用户与电子邮件系统交互的接口，也就是我们日常使用的邮件客户端软件，如 Outlook、Apple Mail 或 Gmail 的网页界面。用户通过用户代理撰写邮件、阅读邮件、管理邮箱。用户代理将编写好的邮件发送到邮件服务器，并从邮件服务器获取收到的邮件。

### 3.2. 邮件服务器

邮件服务器是电子邮件系统的核心。每个接收方都在某个邮件服务器上拥有一个邮箱，邮箱中保存了发给该用户的所有邮件。当发送方的邮件服务器无法将邮件投递到接收方的邮件服务器时，会将邮件放入报文队列中等待重试，通常每隔 30 分钟左右重试一次，如果几天后仍然失败，邮件会被退回给发送方。

### 3.3. 简单邮件传输协议 SMTP

SMTP 是因特网电子邮件系统中用于发送邮件的核心协议。它使用 TCP 协议在发送方的邮件服务器和接收方的邮件服务器之间建立可靠连接，默认端口号为 25。SMTP 采用推（push）的方式工作——发送方将邮件推送到接收方的服务器，这与 HTTP 使用的拉（pull）方式不同。

## 4. SMTP

### 4.1. SMTP 的基本工作

SMTP 传输邮件的过程分为三个阶段。首先是握手阶段，发送方服务器与接收方服务器建立 TCP 连接，交换身份信息。然后是邮件传输阶段，发送方通过 MAIL FROM 命令指定发件人，RCPT TO 命令指定收件人，DATA 命令发送邮件内容，邮件内容以一个单独的句点（.）结束。最后是关闭阶段，发送 QUIT 命令关闭连接。如果一封邮件有多个收件人，SMTP 会在同一个连接上为每个收件人重复传输过程。

### 4.2. SMTP 与 HTTP 的对比

SMTP 和 HTTP 都是用于从一台主机向另一台主机传送文件的协议，但有显著区别。SMTP 是推协议，发送方将文件推送到接收方；HTTP 是拉协议，客户端从服务器拉取文件。SMTP 要求邮件内容采用 7 比特 ASCII 编码，如果包含非文本内容需要先进行编码；HTTP 没有这个限制。SMTP 将所有内容放在一个报文中一次性传输；HTTP 每个对象使用独立的请求/响应对。

## 5. 邮件报文格式

邮件报文由首部和体两部分组成，两者之间用一个空行分隔。首部包含各种头部字段：From（发件人地址）、To（收件人地址）、Subject（邮件主题）、Date（发送日期）等。邮件体是实际要发送的内容，通常以 7 比特 ASCII 编码。如果邮件包含非 ASCII 内容（如图片、附件），需要使用 MIME（多用途因特网邮件扩展）进行编码，将二进制数据转换为 ASCII 文本格式。

## 6. 邮件访问协议

### 6.1. POP3

POP3（邮局协议版本 3）是最简单的邮件访问协议，使用 TCP 连接，默认端口为 110。POP3 的工作过程分为三个阶段：授权阶段（用户提供用户名和密码）、事务处理阶段（获取邮件列表、下载邮件、标记删除）和更新阶段（退出时服务器删除被标记的邮件）。POP3 的一个主要特点是它不支持在服务器上管理邮件——用户可以选择下载后删除服务器上的邮件或保留邮件，但无法在服务器上创建文件夹或只下载邮件的部分内容。

### 6.2. IMAP

IMAP（因特网邮件访问协议）是功能更强大的邮件访问协议，默认端口为 143。与 POP3 不同，IMAP 让用户能够在服务器上组织和管理邮件——创建文件夹、移动邮件、标记已读等。IMAP 还支持只获取邮件的首部信息而不下载整个邮件体，方便用户快速浏览大量邮件。邮件始终保存在服务器上，用户可以从任何设备访问完整的邮箱内容，这使得 IMAP 成为多设备场景下的首选协议。

### 6.3. 基于 Web 的电子邮件

如今大多数人通过 Web 浏览器来收发邮件，如 Gmail、Outlook.com 等。这种方式表面上不涉及 POP3 或 IMAP 协议，但实际上邮件的发送仍然依赖 SMTP。Web 邮件服务在服务器端处理了邮件的发送和存储逻辑，用户的浏览器通过 HTTP/HTTPS 与 Web 服务器交互，Web 服务器再通过 SMTP 将邮件发送到目标邮件服务器。
