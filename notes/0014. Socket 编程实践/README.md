## 3. UDP 套接字编程

### 3.1. UDP 套接字的基本流程

UDP 是无连接的传输层协议，因此 UDP 套接字编程不需要经历连接建立和连接关闭的过程。发送端的基本流程是：创建套接字（指定 SOCK_DGRAM 类型）、将数据和目标地址封装到数据报中、通过 sendto 函数发送。接收端的基本流程是：创建套接字、绑定本地地址和端口（bind）、通过 recvfrom 函数接收数据报。recvfrom 会阻塞直到有数据到达，并同时返回发送方的地址信息。

### 3.2. 发送端程序

UDP 发送端程序的核心步骤是创建数据报套接字，构造包含目标 IP 地址和端口号的地址结构，然后调用 sendto 将消息发送出去。发送的数据被封装在一个 UDP 报文段中，加上源端口号、目的端口号、长度和校验和等首部信息后交给 IP 层。由于 UDP 是无连接的，sendto 不需要事先建立连接，每次发送都可以指定不同的目标地址。

### 3.3. 接收端程序

UDP 接收端程序需要先创建套接字，然后通过 bind 将套接字与本地的 IP 地址和端口号绑定，这样发送端才能知道将数据发往何处。接着进入一个循环，反复调用 recvfrom 等待数据到达。recvfrom 是一个阻塞调用，它会暂停程序执行直到收到数据报。收到数据后，recvfrom 同时返回发送方的地址信息，接收端可以据此回复数据。

## 4. TCP 套接字编程

### 4.1. TCP 套接字编程的基本流程

TCP 是面向连接的协议，因此 TCP 套接字编程需要经历完整的连接建立和关闭过程。服务器端的流程是：创建套接字（SOCK_STREAM 类型）、绑定地址和端口、监听连接（listen）、接受连接（accept）、收发数据（read/write）、关闭连接。客户端的流程是：创建套接字、发起连接（connect）、收发数据、关闭连接。三次握手发生在 connect 和 accept 之间。

### 4.2. 服务器端程序

服务器端程序首先创建一个 TCP 套接字，绑定到指定的 IP 地址和端口号，然后调用 listen 将套接字设置为监听状态，表示它准备好接受连接请求。listen 的参数指定了等待队列的最大长度。当客户端发起连接时，服务器调用 accept 接受连接，accept 会创建一个新的套接字专门用于与该客户端的通信，原来的监听套接字继续等待其他客户端的连接。之后服务器通过新的套接字与客户端交换数据。

### 4.3. 客户端程序

客户端程序创建 TCP 套接字后，不需要绑定地址（操作系统会自动分配一个临时端口），而是直接调用 connect 向服务器发起连接。connect 需要传入服务器的 IP 地址和端口号，它会触发 TCP 三次握手。连接建立后，客户端通过 write 发送数据、通过 read 接收响应。通信完成后调用 close 关闭连接，触发 TCP 的四次挥手过程。
