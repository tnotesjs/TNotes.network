## 3. 运输层和网络层的关系

### 3.1. 运输层协议的作用

运输层位于协议栈的第四层，负责在两个端系统的不同进程之间提供逻辑通信。网络层提供了主机到主机的通信服务，但一台主机上可能同时运行着多个应用程序，运输层需要将数据准确地交付给正确的应用进程。运输层协议接收应用层的消息，将其分割为报文段，添加运输层首部（包含端口号等信息），然后交给网络层发送。

### 3.2. 因特网运输层概述

因特网提供两种运输层协议：TCP 和 UDP。TCP 是面向连接的可靠传输协议，提供流量控制、拥塞控制、数据完整性保障和有序交付。UDP 是无连接的轻量级协议，只提供多路复用/多路分解和少量差错检测功能，不保证可靠性。选择哪种协议取决于应用的需求——Web、电子邮件、文件传输等需要可靠性的应用使用 TCP，而视频流、DNS 查询、实时游戏等对延迟敏感的应用可以使用 UDP。

## 4. 多路复用与多路分解

### 4.1. 什么是多路复用与多路分解

多路复用是指在发送端，运输层从多个套接字收集数据，为每个数据块添加首部信息（包含端口号等标识），然后将它们交给网络层。多路分解是指在接收端，运输层从网络层收到报文段后，根据首部中的端口号将数据定向到正确的套接字。这两个过程使得多个应用可以共享同一个网络连接。

### 4.2. 无连接的多路复用与多路分解

UDP 使用二元组（目的 IP 地址、目的端口号）来标识一个套接字。当一个 UDP 报文段到达主机时，运输层检查报文段中的目的端口号，将其定向到具有该端口号的套接字。来自不同源地址和源端口但目的端口相同的报文段，会被定向到同一个套接字。

### 4.3. 面向连接的多路复用与多路分解

TCP 使用四元组（源 IP 地址、源端口号、目的 IP 地址、目的端口号）来唯一标识一个连接。这意味着两个不同的客户端可以同时连接到同一个服务器的同一个端口，因为它们的源地址或源端口不同，四元组不会重复。服务器为每个接受的 TCP 连接创建一个独立的套接字，由对应的四元组来标识。

### 4.4. Web 服务器与 TCP

Web 服务器通常监听在 80 端口（HTTP）或 443 端口（HTTPS）。当多个客户端同时访问同一个 Web 服务器时，服务器通过四元组区分每个连接。Web 服务器可以为每个连接创建一个新线程或进程来处理，也可以使用异步 I/O 在单个线程中处理多个连接。持久连接（HTTP/1.1 默认）下，同一个客户端的多个请求可以通过同一个 TCP 连接发送。
