## 3. TCP 连接

### 3.1. 什么是 TCP 连接

TCP 是面向连接的协议，在发送数据之前，通信双方必须先建立一条逻辑连接。这条连接由两个端系统的 TCP 协议栈共同维护，包括双方的发送和接收缓冲区、各种状态变量（如序号、窗口大小）等。TCP 连接是点对点的，即一条连接只能连接两个端点，不支持一对多的广播或多播。

### 3.2. TCP 连接的特点

TCP 连接有几个重要特点。首先是全双工通信——数据可以同时在两个方向上传输。其次是面向字节流——TCP 把数据看作无结构的字节流，序号是按字节编号的而不是按报文段编号。第三是可靠传输——通过序号、确认号和重传机制保证数据无差错、不丢失、不重复且按序到达。

## 4. TCP 报文段结构

### 4.1. 报文段头部

TCP 报文段由首部和载荷两部分组成。首部的固定部分为 20 字节，包含源端口号（16 位）、目的端口号（16 位）、序号（32 位）、确认号（32 位）、首部长度（4 位）、保留字段（6 位）、标志位（6 位）、接收窗口大小（16 位）、检验和（16 位）和紧急数据指针（16 位）。首部长度字段指示首部有多少个 4 字节的字，使得首部长度可以是 20 到 60 字节之间的任意 4 的倍数。

### 4.2. 序号和确认号

序号是 TCP 可靠传输的基础。TCP 把数据看作一个无结构的字节流，序号字段的值是该报文段中第一个数据字节在字节流中的编号。例如第一个报文段携带 500 字节数据，序号为 0，则第二个报文段的序号为 500。

确认号是接收方期望收到的下一个字节的编号。当接收方收到序号为 0、包含 500 字节的报文段后，它会发送一个确认号为 500 的 ACK，表示「我已收到 0 到 499 字节，期望从 500 开始的下一个字节」。TCP 采用累积确认——确认号 $N$ 表示序号 $N$ 之前的所有字节都已正确收到。

### 4.3. 标志字段

TCP 首部中有 6 个标志位，各占 1 位。SYN 用于连接建立阶段，表示请求同步序号。FIN 用于连接关闭阶段，表示发送方数据已发送完毕。RST 用于重置连接，通常在出现异常时使用。ACK 标志确认号字段有效，除了第一个 SYN 报文段外，所有报文段都应设置 ACK。PSH 表示接收方应尽快将数据交付给应用层。URG 表示报文段中有紧急数据。

## 5. 往返时间的估计与超时

### 5.1. 估计往返时间

TCP 需要估计报文段的往返时间（RTT）来设置超时重传的时间间隔。TCP 通过测量从发送报文段到收到对应 ACK 的时间来采样 RTT。由于网络延迟存在波动，TCP 使用加权移动平均来更新估计值：

$$EstimatedRTT = (1 - \alpha) \times EstimatedRTT + \alpha \times SampleRTT$$

其中 $\alpha$ 通常取 0.125。同时 TCP 还计算 RTT 的偏差（DevRTT），用于衡量采样值的波动程度，以便在估计值的基础上添加适当的安全余量。

### 5.2. 设置和管理重传超时间隔

重传超时间隔（TimeoutInterval）的设置需要在两个目标之间平衡：既不能太短导致不必要的重传，也不能太长导致丢包后等待过久。TCP 将超时间隔设置为：

$$TimeoutInterval = EstimatedRTT + 4 \times DevRTT$$

初始值设为 1 秒。如果发生超时重传，TCP 会将超时间隔加倍（指数退避），避免在网络拥塞时持续加重负担。每次成功收到 ACK 后，超时间隔会重新计算。

## 6. TCP 连接管理

### 6.1. 三次握手建立连接

TCP 使用三次握手来建立连接。第一步，客户端发送一个 SYN 报文段，其中 SYN 标志位置 1，并携带客户端选择的初始序号。第二步，服务器收到 SYN 后，回复一个 SYN+ACK 报文段，其中 SYN 和 ACK 标志位都置 1，并携带服务器选择的初始序号和对客户端序号的确认。第三步，客户端收到 SYN+ACK 后，发送一个 ACK 报文段确认服务器的序号。至此双方确认了彼此的初始序号，连接建立完成。

### 6.2. 四次挥手关闭连接

TCP 使用四次挥手来关闭连接。首先，客户端发送一个 FIN 报文段，表示客户端没有更多数据要发送了。服务器收到后回复 ACK，此时连接处于半关闭状态——服务器仍可以继续发送数据。当服务器也没有更多数据要发送时，发送自己的 FIN 报文段。客户端收到后回复 ACK，并进入 TIME_WAIT 状态，等待一段时间（通常为 2MSL，即两倍的最大报文段生存时间）后彻底关闭连接。TIME_WAIT 状态的目的是确保最后的 ACK 能够到达服务器，以及让网络中残余的报文段过期。

### 6.3. 连接状态

TCP 连接在其生命周期中会经历多个状态。客户端的状态转换通常是：CLOSED → SYN_SENT → ESTABLISHED → FIN_WAIT_1 → FIN_WAIT_2 → TIME_WAIT → CLOSED。服务器的状态转换通常是：CLOSED → LISTEN → SYN_RCVD → ESTABLISHED → CLOSE_WAIT → LAST_ACK → CLOSED。每个状态都有明确的含义，比如 LISTEN 表示服务器正在等待客户端的连接请求，ESTABLISHED 表示连接已建立可以传输数据。
