## 3. 转发和路由选择

### 3.1. 数据平面

数据平面负责路由器的本地操作——当一个分组到达路由器的输入端口时，路由器通过查找转发表决定将其从哪个输出端口转发出去。这个转发过程是每个分组都要执行的，要求极高的速度。现代路由器通常使用硬件（如 TCAM）来实现转发表查找，以达到线速转发的能力。数据平面的操作是完全在单台路由器内完成的。

### 3.2. 控制平面

控制平面负责网络范围内的逻辑——确定分组从源到目的地应该走哪条路径。路由选择算法是控制平面的核心，它根据网络拓扑和链路状态计算出最优路径，并将结果写入每台路由器的转发表。控制平面可以是传统的分布式方式（每台路由器运行路由协议如 OSPF、BGP），也可以是集中式的 SDN 方式（由中央控制器统一计算路由）。

### 3.3. 数据平面与控制平面的关系

数据平面和控制平面是紧密配合的。控制平面通过路由选择算法计算出转发表，数据平面根据转发表快速转发分组。可以类比为：控制平面是「规划路线的导航系统」，数据平面是「按照路线行驶的司机」。在传统路由器中，这两个平面都在同一台设备上。在 SDN 架构中，控制平面被分离到独立的控制器上，路由器只负责数据平面的转发操作。

## 4. 网络服务模型

### 4.1. 确保交付

网络服务模型定义了网络层对上层提供的服务承诺。最基本的承诺是确保交付——保证分组最终能到达目的地，不丢失。这是一种非常基本的服务保证，实际的因特网 IP 协议甚至不提供这个保证（它是尽力而为的服务）。

### 4.2. 具有时延上界的确保交付

更强的服务模型会在确保交付的基础上增加时延保证——不仅保证分组能到达，还保证在指定的时间上限内到达。这对实时应用（如视频会议、远程手术）非常重要，但因特网的 IP 协议不提供时延保证。

### 4.3. 有序分组交付

有序分组交付保证分组按照发送顺序到达目的地。乱序到达的分组需要在接收端重新排序，这增加了复杂性和延迟。TCP 在运输层提供了有序交付，但 IP 层本身不保证顺序。

### 4.4. 最小带宽保证

最强的服务模型之一是提供最小带宽保证——确保发送方能够以某个最低速率发送数据。这类似于电路交换中预留带宽的思想，对带宽敏感的应用（如高清视频流）非常有用。因特网不提供带宽保证，但可以通过服务质量（QoS）机制来近似实现。
