## 3. 输入端口处理和基于目的地转发

### 3.1. 输入端口处理

分组从某条输入链路到达路由器后，首先在输入端口进行处理。输入端口的功能包括：物理层的线路终结（将信号转换为比特）、链路层的解封装（提取网络层数据报）、查找转发表决定输出端口、将分组送到交换结构。其中查找转发表是最关键的操作，需要在极短的时间内完成以避免成为瓶颈。

### 3.2. 基于目的地的转发

传统的路由器采用基于目的地的转发——只根据分组的目的 IP 地址来查找转发表，确定输出端口。转发表中存储了目的地址前缀到输出链路的映射。路由器将分组的目的地址与转发表中的条目逐一比较，找到匹配项后将分组转发到对应的输出端口。

### 3.3. 最长前缀匹配

转发表中的目的地址通常以 CIDR 前缀的形式表示，如 223.1.1.0/24。一个分组的目的地址可能匹配多个前缀（例如同时匹配 223.1.1.0/24 和 223.1.0.0/16），此时路由器选择最长的匹配前缀。最长前缀匹配是因特网路由的核心机制，它使得路由表可以灵活地表示各种大小的地址块。现代路由器使用三态内容可寻址存储器（TCAM）来硬件实现最长前缀匹配。

## 4. 交换

### 4.1. 经内存交换

最早的路由器使用内存交换方式，类似于传统的计算机 I/O 操作。输入端口将分组通过系统总线写入内存，处理器查找转发表后将分组从内存读出到输出端口。这种方式的交换速率受限于内存的读写带宽和系统总线的速度。

### 4.2. 经总线交换

经总线交换中，输入端口通过一条共享总线直接将分组传送到输出端口，无需经过处理器和内存。分组在总线上传输时，所有输出端口都能收到，但只有目的端口会接收。总线的带宽成为瓶颈——如果多个输入端口同时要传送分组到不同的输出端口，它们必须分时共享总线。

### 4.3. 经互连网络交换

现代高端路由器使用互连网络（如纵横式交换机）来实现并行交换。互连网络由多条交叉的路径组成，可以同时在多个输入端口和输出端口之间建立独立的连接。这意味着来自不同输入端口的分组可以同时传送到不同的输出端口，消除了总线交换的带宽瓶颈。但如果两个输入端口同时要向同一个输出端口发送分组，仍然会发生竞争。

## 5. 输出端口处理

输出端口负责将分组发送到输出链路上。它从交换结构接收分组，将其放入输出缓存的队列中，然后按照选定的调度策略从队列中取出分组，进行链路层和物理层的封装后发送出去。输出端口处理的速率必须与链路的传输速率匹配，否则队列会不断增长。

## 6. 排队与分组调度

### 6.1. 输入排队

当交换结构的速率慢于输入链路的总速率时，分组会在输入端口排队等待。输入排队存在线路前部（HOL）阻塞问题——队列头部的分组因为要去的输出端口正忙而被阻塞，但它后面的分组即使要去空闲的输出端口也必须等待。HOL 阻塞会显著降低交换结构的吞吐量。

### 6.2. 输出排队

当多个输入端口同时要向同一个输出端口发送分组时，分组必须在输出端口排队。输出排队是最常见的排队场景。输出缓存的大小直接影响丢包率和排队时延。实践中通常使用「经验法则」来设置缓存大小，即缓存大小等于平均往返时延乘以链路容量。

### 6.3. 分组调度策略

调度策略决定了队列中的分组以什么顺序被发送。先进先出（FIFO）是最简单的策略，按到达顺序发送。优先权排队为不同类型的分组设置不同的优先级，高优先级的分组总是先发送。加权公平排队（WFQ）为每种类型的分组分配一定的权重，确保每种类型都能获得一定比例的带宽。循环调度则在多个队列之间轮流服务。
