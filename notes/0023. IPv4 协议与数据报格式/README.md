## 3. IPv4 数据报格式

### 3.1. 首部各字段的含义

IPv4 数据报的首部通常为 20 字节（无选项字段时），包含多个重要字段。版本号（4 位）标识 IP 协议版本，IPv4 中为 4。首部长度（4 位）以 4 字节为单位表示首部长度。服务类型（8 位）用于区分不同类型的分组。总长度（16 位）表示整个数据报的字节数，最大为 65535 字节。标识（16 位）、标志位（3 位）和片偏移（13 位）共同用于数据报分片和重组。生存时间 TTL（8 位）限制数据报在网络中的存活时间，每经过一台路由器减 1，减到 0 时丢弃。协议（8 位）标识载荷属于哪个上层协议（如 TCP 为 6，UDP 为 17）。首部检验和（16 位）用于检测首部的比特错误。源 IP 地址和目的 IP 地址各占 32 位。

### 3.2. 数据报分片

不同类型的链路层协议对帧的最大载荷有不同的限制，这个限制称为最大传输单元（MTU）。以太网的 MTU 为 1500 字节。当一个 IP 数据报的大小超过链路的 MTU 时，路由器必须将其分片——将数据报拆分成多个较小的片，每个片都是一个独立的 IP 数据报。分片在目的地重组，接收端根据标识、标志位和片偏移字段将所有片按正确顺序组装成原始数据报。如果任何一个片丢失，整个数据报都需要重传。

## 4. IPv4 数据报的转发过程

### 4.1. 直接交付

直接交付是指目的主机与发送方在同一个子网中，发送方可以直接将数据报通过链路层发送到目的主机，无需经过路由器。发送方通过将目的 IP 地址与自己的子网掩码进行按位与运算，判断目的地址是否在自己的子网内。如果在，就使用 ARP 协议获取目的主机的 MAC 地址，直接发送。

### 4.2. 间接交付

间接交付是指目的主机不在发送方的子网中，数据报必须通过一台或多台路由器转发。发送方将数据报发送到默认网关（通常是连接本子网的路由器），路由器收到后查找转发表，确定下一跳的 IP 地址，再通过链路层将数据报转发到下一跳。这个过程逐跳进行，直到数据报到达目的主机所在的子网，由最后一跳路由器进行直接交付。
