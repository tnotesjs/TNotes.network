## 3. IPv4 编址

### 3.1. IP 地址的基本格式

IPv4 地址是一个 32 位的二进制数，通常用点分十进制表示——将 32 位分为 4 个字节，每个字节用十进制表示，中间用点分隔。例如 192.168.1.1 的二进制形式是 11000000.10101000.00000001.00000001。IP 地址是接口的标识而非主机的标识——一台有多个网络接口的主机（如同时连接有线网和 WiFi）会有多个 IP 地址，每个接口一个。

### 3.2. 特殊 IP 地址

有一些 IP 地址具有特殊含义。255.255.255.255 是受限广播地址，发送到这个地址的分组会被本子网内的所有主机接收，路由器不会转发。0.0.0.0 在某些上下文中表示「本机」。127.0.0.1 是回环地址，发送到这个地址的分组会直接返回本机，用于本地测试。以 10.、172.16.到 172.31.、192.168.开头的地址是私有地址，不能在公网上使用，只在局域网内部有效。

## 4. 子网划分

### 4.1. 子网的概念

IP 地址由两部分组成：网络部分（高位）和主机部分（低位）。具有相同网络部分的接口属于同一个子网，它们之间可以直接通信，无需经过路由器。子网划分将一个大的 IP 地址空间分割成多个较小的子网，每个子网通过路由器相互连接。这种层次化的地址结构使得路由表可以大幅压缩——路由器只需知道如何到达每个子网，而不是每个单独的主机。

### 4.2. 子网掩码

子网掩码用来指示 IP 地址中哪些位是网络部分、哪些位是主机部分。掩码中为 1 的位对应网络部分，为 0 的位对应主机部分。例如子网掩码 255.255.255.0 表示前 24 位是网络部分，后 8 位是主机部分。要判断两个 IP 地址是否在同一个子网中，只需将各自的 IP 地址与子网掩码进行按位与运算，如果结果相同就在同一个子网。

### 4.3. CIDR

CIDR（无类别域间路由）打破了传统的 A、B、C 类地址划分，允许使用任意长度的前缀来表示网络部分。CIDR 表示法为「IP地址/前缀长度」，如 223.1.1.0/24 表示前 24 位是网络部分。CIDR 使得地址分配更加灵活高效，减少了地址浪费。CIDR 还支持路由聚合——多个连续的子网可以聚合成一个更大的地址块，减少了路由表的条目数量。

## 5. DHCP

### 5.1. DHCP 的工作原理

DHCP（动态主机配置协议）允许主机自动获取 IP 地址和其他网络配置参数（如子网掩码、默认网关、DNS 服务器地址）。DHCP 的工作过程分为四步（DORA）：客户端广播 DHCP DISCOVER 报文寻找 DHCP 服务器；服务器回复 DHCP OFFER 报文提供一个 IP 地址；客户端广播 DHCP REQUEST 报文接受该地址；服务器回复 DHCP ACK 报文确认分配。DHCP 使得新设备接入网络时无需手动配置 IP 地址。

### 5.2. DHCP 的报文格式

DHCP 报文基于 UDP 协议传输，客户端使用端口 68，服务器使用端口 67。报文包含客户端的 MAC 地址、请求的 IP 地址、分配的 IP 地址、子网掩码、默认网关、DNS 服务器地址等字段。IP 地址的分配是有时间限制的——DHCP 服务器为每个分配的地址设置一个租期，客户端需要在租期到期前续租，否则地址会被回收。
