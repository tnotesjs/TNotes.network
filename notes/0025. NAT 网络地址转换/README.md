## 3. NAT 的背景与动机

### 3.1. IPv4 地址耗尽问题

IPv4 地址只有 32 位，理论上最多只能表示约 43 亿个地址。随着互联网的快速发展，可用的公网 IPv4 地址早已耗尽。然而大量家庭和小型企业的网络中只有少量设备需要接入互联网，为每台设备分配一个公网 IP 地址既不现实也不经济。NAT 的出现正是为了解决这个矛盾。

### 3.2. NAT 的基本思想

NAT 的核心思想是：一个局域网内的所有设备共享一个公网 IP 地址来访问互联网。NAT 路由器位于局域网和互联网之间，负责将内部设备的私有 IP 地址转换为公网 IP 地址。这样即使局域网内有几十台设备，对外也只需要一个公网 IP 地址。

## 4. NAT 的工作原理

### 4.1. NAT 转换表

NAT 路由器维护一张 NAT 转换表，记录内部地址和外部地址之间的映射关系。表中的每一行包含一个映射条目，将（内部源 IP 地址、内部源端口号）映射到（外部源 IP 地址、外部源端口号）。通过使用端口号作为区分标识，NAT 可以让局域网内的多个设备共享同一个公网 IP 地址的不同端口。

### 4.2. NAT 的工作流程

当内部主机发送一个数据包到外部网络时，NAT 路由器拦截该数据包，将源 IP 地址和源端口号替换为公网 IP 地址和一个未使用的端口号，并在转换表中记录这个映射。当外部服务器的响应到达 NAT 路由器时，路由器查找转换表，根据目的端口号找到对应的内部主机地址，将目的地址替换为内部地址后转发给正确的主机。

## 5. NAT 的优缺点

### 5.1. NAT 的优点

NAT 最大的优点是节约了公网 IPv4 地址——只需要一个公网 IP 地址就能让整个局域网接入互联网。此外，NAT 还提供了一定的安全性，因为外部网络无法直接访问内部主机的私有 IP 地址，NAT 路由器起到了天然的防火墙作用。

### 5.2. NAT 的缺点

NAT 的主要缺点是违反了 IP 协议的端到端原则——外部主机无法主动发起与内部主机的连接，因为内部主机的私有地址对外部是不可见的。这对 P2P 应用、VoIP 等需要双向通信的场景造成了困扰。NAT 还增加了网络的复杂性，某些协议（如 FTP、SIP）在载荷中嵌入了 IP 地址信息，NAT 需要额外的应用层网关来处理这些情况。

### 5.3. NAT 穿透

NAT 穿透是指让外部主机能够与 NAT 后面的内部主机建立连接的技术。常见的方法包括：端口映射（在 NAT 路由器上手动配置端口转发规则）、STUN 协议（让内部主机通过 STUN 服务器发现自己的公网地址和端口）、中继服务器（无法直接连接时通过第三方服务器中继数据）。这些技术在实际应用中被广泛使用，如视频通话、在线游戏等场景。
