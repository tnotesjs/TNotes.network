## 3. IPv6 数据报格式

### 3.1. 首部各字段的含义

IPv6 数据报的首部固定为 40 字节，比 IPv4 的首部更加简洁。版本号（4 位）标识为 6。流量类别（8 位）类似于 IPv4 的服务类型。流标签（20 位）用于标识同一个数据流，路由器可以对同一流的分组进行特殊处理。载荷长度（16 位）表示首部之后的载荷长度。下一个首部（8 位）标识载荷属于哪个上层协议或扩展首部。跳数限制（8 位）相当于 IPv4 的 TTL。源地址和目的地址各占 128 位。

### 3.2. 与 IPv4 的区别

IPv6 相比 IPv4 有几个显著变化。首先地址从 32 位扩展到 128 位，地址空间从约 43 亿增加到约 3.4×10^38，从根本上解决了地址耗尽问题。其次首部简化了——取消了首部检验和（由上层协议保证）、取消了分片相关字段（只在源端分片，路由器不再分片）。IPv6 不支持广播，使用多播和任播替代。IPv6 还取消了 NAT 的必要性，因为地址足够充裕。

## 4. IPv6 编址

### 4.1. IPv6 地址格式

IPv6 地址用冒号分隔的十六进制表示，共 8 组，每组 4 个十六进制数字，如 2001:0db8:85a3:0000:0000:8a2e:0370:7334。为了简化书写，可以省略每组前导的零（如 0db8 写成 db8），也可以用双冒号（::）替代连续的全零组（但一个地址中只能使用一次双冒号）。

### 4.2. IPv6 地址类型

IPv6 有三种主要的地址类型。单播地址标识单个接口，发送到单播地址的分组只到达一个接口。多播地址标识一组接口，分组被交付给该组中的所有接口。任播地址也标识一组接口，但分组只被交付给该组中距离最近的一个接口。IPv6 没有广播地址，广播的功能由多播地址（如 ff02::1 表示链路上的所有节点）替代。

## 5. 从 IPv4 到 IPv6 的迁移

### 5.1. 双栈

双栈是最直接的迁移方法——让网络中的所有设备同时运行 IPv4 和 IPv6 两个协议栈。当两个 IPv6 设备通信时使用 IPv6，当需要与 IPv4 设备通信时回退到 IPv4。双栈的缺点是需要同时维护两套协议，增加了设备的复杂性和管理成本。

### 5.2. 隧道

隧道技术用于在 IPv4 网络中传输 IPv6 分组。当两个 IPv6 节点之间只存在 IPv4 路径时，发送端的隧道入口将整个 IPv6 分组封装在 IPv4 分组的载荷中，通过 IPv4 网络传输到隧道出口，出口再将 IPv6 分组解封装出来。6to4、6in4、Teredo 等都是常见的隧道技术。隧道的缺点是增加了额外的头部开销和配置复杂性。
