## 3. 因特网中自治系统内部的路由选择——OSPF

### 3.1. 自治系统的概念

因特网的规模太大，单一的路由算法无法处理所有路由器之间的路由选择。因此因特网将路由器划分为多个自治系统（AS），每个 AS 是一个由单一组织管理的网络，包含一组在统一路由策略下运行的路由器。AS 内部的路由选择使用内部网关协议（IGP），不同 AS 之间的路由选择使用外部网关协议（EGP）。

### 3.2. OSPF 的基本原理

OSPF（开放最短路径优先）是因特网中最广泛使用的内部网关协议。OSPF 是一种链路状态协议——每台路由器向 AS 内的所有其他路由器洪泛自己的链路状态信息（连接的邻居和链路代价）。每台路由器收到完整的链路状态信息后，构建网络拓扑图并运行 Dijkstra 算法计算最短路径。OSPF 使用链路代价（通常与链路带宽成反比）作为度量值。

### 3.3. OSPF 的区域

为了减少链路状态洪泛的范围和计算开销，OSPF 支持将 AS 划分为多个区域。每个区域运行独立的链路状态算法，路由器只在自己所属的区域内洪泛链路状态信息。区域之间通过区域边界路由器连接，骨干区域（Area 0）负责连接所有其他区域。这种层次化的结构使得 OSPF 能够扩展到大型网络。

## 4. ISP 之间的路由选择——BGP

### 4.1. BGP 的作用

BGP（边界网关协议）是因特网中唯一的自治系统间路由协议。BGP 不仅负责在 AS 之间交换路由信息，还负责实施路由策略——决定哪些路由可以接受、哪些可以通告、哪些应该优先选择。BGP 是因特网能够正常运转的关键协议，被称为「将因特网粘合在一起的协议」。

### 4.2. 通告 BGP 路由信息

BGP 使用 TCP 连接在对等体（peer）之间交换路由信息。eBGP 用于不同 AS 之间的路由交换，iBGP 用于同一 AS 内部的路由器之间传递从外部学到的路由。BGP 通告的路由不仅包含目的网络的前缀，还包含完整的 AS 路径——路由经过的所有 AS 的序列。接收方通过检查 AS 路径可以检测路由环路（如果自己的 AS 号已经出现在路径中，就丢弃该路由）。

### 4.3. 确定最好的路由

当一个 BGP 路由器收到多条到达同一目的地的路由时，需要按照优先级选择最好的路由。BGP 的路由选择过程依次考虑以下因素：本地偏好值（由管理员设置的策略值）、最短的 AS 路径、最近的下一跳路由器、其他 tie-breaker 规则。本地偏好值的优先级最高，这使得 ISP 可以通过设置偏好值来控制流量的走向，实现商业策略。

### 4.4. IP 任播

BGP 支持 IP 任播技术——多个不同地理位置的服务器使用同一个 IP 地址。当路由器收到多条到达同一任播地址的路由时，会选择最短 AS 路径的那条。用户的请求会被自动路由到最近的服务器。IP 任播被广泛用于 DNS 根服务器的部署——13 组根服务器实际上由数百台物理服务器通过任播技术分布在全球各地。

### 4.5. 路由选择策略

BGP 的路由选择策略体现了 ISP 之间的商业关系。主要有三种关系：客户-提供商（客户付费使用提供商的网络）、对等（两个 ISP 免费相互交换流量）、兄弟（同一组织下的 ISP 之间免费交换）。基本策略是：不转发从客户学到的路由给提供商（避免成为免费的中转），优先选择通过客户路径到达目的地（因为这是付费路径）。
