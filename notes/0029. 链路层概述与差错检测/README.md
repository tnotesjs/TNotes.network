## 3. 链路层概述

### 3.1. 链路层提供的服务

链路层负责将数据报从一个节点通过单一通信链路传送到相邻节点。它提供的服务包括成帧——在数据报的前后添加首部和尾部形成帧；链路接入——控制对共享媒体的访问；可靠交付——保证数据报无差错地通过链路（某些链路协议如无线链路提供此服务，以太网不提供）；差错检测和纠正——检测或纠正帧在传输中出现的比特错误。

### 3.2. 链路层在何处实现

链路层协议主要在网络适配器（也叫网络接口卡 NIC）中实现。NIC 通常是一块独立的硬件板卡（或集成在主板上），包含链路层控制器芯片、物理层接口等。NIC 通过总线与主机的 CPU 和内存连接。当发送数据时，CPU 将数据报写入 NIC 的缓存，NIC 负责添加链路层首部和尾部并通过物理媒体发送。当接收数据时，NIC 检验帧的正确性，提取数据报并通过中断通知 CPU。

## 4. 差错检测和纠正技术

### 4.1. 奇偶校验

最简单的差错检测方法是奇偶校验。在单比特奇偶校验中，发送方在数据后附加一个校验位，使得整个数据（包括校验位）中 1 的个数为偶数（偶校验）或奇数（奇校验）。接收方检查 1 的个数是否符合预期，如果不符合就说明发生了错误。单比特奇偶校验只能检测奇数个比特错误，偶数个比特错误无法检测到。二维奇偶校验通过将数据排列成矩阵，在行和列上分别计算校验位，可以检测并纠正单比特错误。

### 4.2. 检验和方法

检验和方法将数据视为 16 比特整数的序列，对它们求和并对结果取反码。接收方用同样的方法计算数据的检验和，如果结果为 0 则认为没有错误。检验和的优点是计算简单、开销小，但检错能力不如 CRC。IP、TCP 和 UDP 协议都使用检验和方法。

### 4.3. 循环冗余检测（CRC）

CRC 是链路层最常用的差错检测技术，具有更强的检错能力。发送方和接收方预先商定一个 $r+1$ 位的生成多项式 $G$。发送方在 $d$ 位数据后面附加 $r$ 位的 CRC 冗余码，使得整个 $d+r$ 位的比特模式能被 $G$ 整除。接收方将收到的整个比特模式除以 $G$，如果余数为 0 则没有检测到错误；如果余数不为 0 则说明发生了错误。CRC 能检测所有长度不超过 $r$ 的突发错误。
