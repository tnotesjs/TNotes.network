## 3. 链路层寻址和 ARP

### 3.1. MAC 地址

MAC 地址（也叫局域网地址或物理地址）是链路层使用的地址，长度为 48 位（6 字节），通常用十六进制表示，如 1A:2B:3C:4D:5E:6F。MAC 地址由 IEEE 统一管理，每个网络适配器出厂时被分配一个全球唯一的 MAC 地址。与 IP 地址不同，MAC 地址是扁平结构（没有层次），无论设备移动到哪个网络，MAC 地址都不会改变。

### 3.2. ARP 地址解析协议

ARP（地址解析协议）负责将 IP 地址解析为 MAC 地址。当主机知道目的 IP 地址但不知道对应的 MAC 地址时，它在局域网中广播一个 ARP 请求报文，询问「谁拥有这个 IP 地址？」。拥有该 IP 地址的主机回复一个 ARP 应答报文，包含自己的 MAC 地址。发送方将结果缓存在 ARP 表中，后续通信直接查表，避免重复广播。

### 3.3. 发送数据报到子网以外

当目的主机不在同一子网时，发送方需要将帧发送到默认网关（路由器）。发送方通过 ARP 获取默认网关的 MAC 地址，将目的 IP 地址设为最终目的地，但目的 MAC 地址设为网关的 MAC 地址。路由器收到帧后，根据目的 IP 地址查找转发表，确定下一跳，再用 ARP 获取下一跳的 MAC 地址，重新封装帧后转发。每一跳的 MAC 地址都不同，但 IP 地址始终不变。

## 4. 以太网

### 4.1. 以太网帧结构

以太网帧由以下字段组成。前同步码（8 字节）用于接收方的时钟同步。目的 MAC 地址和源 MAC 地址各 6 字节。类型字段（2 字节）标识载荷属于哪个网络层协议（如 0x0800 表示 IPv4）。载荷（46 到 1500 字节）是网络层数据报。CRC（4 字节）用于差错检测。以太网帧的最小长度为 64 字节（不含前同步码），最大长度为 1518 字节。

### 4.2. 以网技术

以太网技术经历了从共享总线到交换式的演进。早期的以太网使用同轴电缆或集线器连接所有节点，采用 CSMA/CD 协议来处理碰撞。现代以太网使用交换机连接节点，每个节点独享一条到交换机的链路，不再需要 CSMA/CD。交换式以太网支持全双工通信，两端可以同时发送和接收数据。

### 4.3. 曼彻斯特编码

以太网使用曼彻斯特编码来传输比特。在这种编码方案中，每个比特周期的中间都有一次电压跳变：从低到高的跳变表示 0，从高到低的跳变表示 1。曼彻斯特编码的优点是接收方可以从信号本身恢复时钟，不需要单独的时钟线。缺点是信号的频率是比特率的两倍，带宽效率较低。

## 5. 链路层交换机

### 5.1. 交换机的转发和过滤

链路层交换机根据帧的目的 MAC 地址来决定转发还是过滤。交换机维护一张交换表，记录 MAC 地址与端口的映射关系。当一个帧到达时，交换机查找交换表：如果目的 MAC 地址对应的端口与帧的入端口相同，就丢弃帧（过滤）；如果不同，就将帧转发到对应的端口；如果找不到对应的条目，就向除入端口外的所有端口泛洪（广播）。

### 5.2. 自学习

交换机的交换表是自动构建的，不需要人工配置。当一个帧到达时，交换机会检查帧的源 MAC 地址，将源 MAC 地址与入端口的映射记录到交换表中。交换表中的每条记录都有一个过期时间，如果一段时间内没有看到某个 MAC 地址的帧，对应的记录会被删除。通过这种自学习机制，交换机可以逐步「学习」到网络中所有设备的位置。

### 5.3. 交换机与路由器的比较

交换机和路由器都是存储转发设备，但工作在不同的层次。交换机工作在链路层，根据 MAC 地址转发帧；路由器工作在网络层，根据 IP 地址转发数据报。交换机的转发表是自动学习的，路由器的转发表是通过路由协议计算的。交换机不隔离广播域（广播帧会被转发到所有端口），路由器隔离广播域。在实际网络中，交换机通常用于局域网内部，路由器用于连接不同的子网。
