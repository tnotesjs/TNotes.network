## 3. ARP 地址解析

### 3.1. ARP 的工作原理

ARP 的工作过程分为四步。当主机 A 需要向同一子网内的主机 B 发送数据，但只知道 B 的 IP 地址时，A 先检查自己的 ARP 缓存中是否有 B 的 MAC 地址。如果没有，A 构造一个 ARP 请求报文，其中包含 A 的 IP 和 MAC 地址以及 B 的 IP 地址，然后将这个报文以广播方式发送到子网中的所有设备。主机 B 发现请求中的目标 IP 是自己的，就回复一个 ARP 应答报文，包含自己的 MAC 地址。其他主机收到请求后忽略。

### 3.2. ARP 报文格式

ARP 报文包含以下字段：硬件类型（如以太网为 1）、协议类型（如 IPv4 为 0x0800）、硬件地址长度（以太网 MAC 地址为 6）、协议地址长度（IPv4 为 4）、操作类型（请求为 1，应答为 2）、发送方 MAC 地址、发送方 IP 地址、目标 MAC 地址、目标 IP 地址。ARP 请求报文中的目标 MAC 地址设为全 0（未知）。ARP 报文直接封装在链路层帧中，不经过 IP 层。

### 3.3. ARP 缓存

主机和路由器都会维护一个 ARP 缓存表，保存最近解析过的 IP-MAC 地址映射。缓存条目有生存时间（通常 20 分钟），过期后会被删除。使用缓存可以避免每次通信都要广播 ARP 请求，显著减少了网络中的广播流量。在 Windows 系统中，可以用 arp -a 命令查看 ARP 缓存。

### 3.4. 跨子网的 ARP

当目的主机不在同一子网时，ARP 的工作方式有所不同。发送方需要获取的不是目的主机的 MAC 地址，而是默认网关（路由器）的 MAC 地址。发送方发起 ARP 请求，询问默认网关的 IP 地址对应的 MAC 地址。获得网关的 MAC 地址后，发送方将数据帧发送给网关，由网关负责转发到目的子网。在目的子网中，路由器再通过 ARP 获取目的主机的 MAC 地址。

## 4. 虚拟局域网 VLAN

### 4.1. 什么是 VLAN

虚拟局域网（VLAN）是一种在物理交换机上创建逻辑上独立的局域网的技术。通过 VLAN，一台物理交换机可以被划分为多个逻辑交换机，每个 VLAN 是一个独立的广播域。属于不同 VLAN 的端口之间不能直接通信，即使它们连接在同一台物理交换机上。VLAN 的好处包括：限制广播域的大小、增强安全性（不同部门之间隔离）、简化网络管理（设备可以在不改变物理连接的情况下切换 VLAN）。

### 4.2. VLAN 的配置

VLAN 的配置方式有两种。基于端口的 VLAN 是最常见的方式，管理员将交换机的每个端口分配到某个 VLAN。基于 MAC 地址的 VLAN 根据设备的 MAC 地址来决定其所属的 VLAN。跨交换机的 VLAN 通过干道链路（trunk）来实现——干道链路上传输的帧会携带 VLAN 标签（802.1Q 标签），接收方交换机根据标签将帧转发到正确的 VLAN 端口。

### 4.3. VLAN 之间的通信

不同 VLAN 之间要通信，必须经过路由器或三层交换机。一种方式是将路由器的每个接口连接到一个 VLAN，但这会消耗大量接口。更常见的做法是使用「单臂路由」——路由器通过一条干道链路连接到交换机，干道链路上承载所有 VLAN 的流量。路由器在不同 VLAN 的子接口之间转发数据报，实现 VLAN 间的路由。
