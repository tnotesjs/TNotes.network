## 3. 设备移动性——网络层视角

### 3.1. 移动性的维度

从网络层的角度看，设备移动性可以分为几个层次。最简单的是在同一子网内移动（如从一个 AP 切换到另一个 AP），此时 IP 地址不变，上层连接不受影响。更复杂的是跨子网移动（如从公司网络移动到家庭网络），此时 IP 地址可能需要改变，上层连接需要某种机制来维持。最复杂的是跨国漫游，涉及不同运营商网络之间的协调。

### 3.2. 移动节点的寻址

当移动设备移动到新的网络时，它需要获得一个新的 IP 地址来在新网络中通信。这个新地址称为转交地址（Care-of Address）。设备可以通过 DHCP 从新网络获得地址，也可以使用外地代理转交地址——由被访网络中的代理服务器为设备提供转交地址。转交地址的用途是让通信对端能够通过转交地址找到移动设备当前所在的位置。

## 4. 归属网络和在被访网络漫游

### 4.1. 归属网络

归属网络是移动设备最初注册的网络，也是设备的永久地址（归属地址）所在的网络。归属网络中有一个归属代理（Home Agent），它负责维护设备当前位置的信息，并在设备离开归属网络时代理转发数据。

### 4.2. 被访网络

被访网络是移动设备当前所在的外部网络。设备在被访网络中获得一个转交地址，并向归属代理注册自己的当前位置。被访网络中可能有一个外地代理（Foreign Agent），帮助设备进行注册和数据转发。

### 4.3. 漫游协议

漫游协议定义了归属网络和被访网络之间的交互规则。当设备移动到被访网络时，它通过被访网络向归属代理发送注册消息，告知自己的转交地址。归属代理收到注册后，建立归属地址和转交地址之间的绑定。之后所有发往设备归属地址的数据包都会被归属代理截获，通过隧道转发到设备的转交地址。

## 5. 去往/来自移动设备的直接和间接路由

### 5.1. 间接路由

在间接路由方案中，通信对端将数据包发送到移动设备的归属地址。归属代理截获这些数据包，通过隧道转发到设备当前的转交地址。设备回复的数据包直接发送给通信对端，不经过归属代理。这种方案的优点是通信对端不需要知道设备的移动性，对通信对端完全透明。缺点是存在「三角路由」问题——数据包要绕道归属网络，增加了延迟。

### 5.2. 直接路由

直接路由方案中，通信对端直接将数据包发送到设备的转交地址，不经过归属代理。通信对端需要先向归属代理查询设备当前的转交地址，然后直接发送数据。直接路由消除了三角路由问题，但增加了复杂性——通信对端需要支持移动性协议，而且设备每次切换网络时，通信对端都需要重新获取新的转交地址。

### 5.3. 两种路由的比较

间接路由对通信对端透明、实现简单，但存在三角路由的延迟和带宽浪费。直接路由消除了三角路由，但增加了实现复杂性。实际系统中通常采用折中方案——初始通信使用间接路由，之后通过优化机制切换到直接路由。

## 6. 4G/5G 网络的移动性管理

### 6.1. 跟踪区域

4G/5G 网络使用跟踪区域（TA）来管理设备的位置。网络将覆盖区域划分为多个 TA，每个 TA 包含一组基站。设备在 TA 内移动时不需要通知网络，只有跨越 TA 边界时才发送位置更新。多个 TA 组成跟踪区域列表（TAI List），设备在列表内的任何 TA 中都是可达的。这种机制减少了位置更新的频率，节省了信令开销。

### 6.2. 切换过程

当移动设备从一个基站移动到另一个基站时，需要进行切换（handover）。切换过程由网络侧控制。当前基站根据设备的测量报告（信号强度、质量等）决定是否需要切换，并选择目标基站。当前基站在目标基提前为设备准备好资源，然后通知设备切换到新基站。设备在新基站上重新建立连接后，数据路径从旧基站切换到新基站。整个过程的目标是尽量减少服务中断时间。
