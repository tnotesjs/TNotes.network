## 3. 对称密钥密码体制

### 3.1. 凯撒密码

凯撒密码是最古老的加密方法之一，原理非常简单——将字母表中的每个字母替换为它后面第 $k$ 个字母。例如 $k=3$ 时，A 变成 D，B 变成 E，以此类推。凯撒密码的安全性极差，因为只有 25 种可能的密钥（$k=1$ 到 $k=25$），可以通过穷举轻松破解。但它展示了密码学的基本思想：通过某种规则将明文转换为密文。

### 3.2. 多码代替密码

多码代替密码是凯撒密码的扩展，使用多个替换规则来增加破解难度。例如 Vigenere 密码使用一个密钥字符串，密钥的每个字母对应一个不同的移位量。第一个明文字母用密钥第一个字母的移位量加密，第二个用第二个，以此循环。多码代替密码比单码密码安全得多，因为同一个明文字母在不同位置会被加密成不同的密文字母。但通过频率分析等方法仍然可以破解。

### 3.3. 块密码

现代对称加密使用块密码——将明文分成固定大小的块（如 128 位），每个块独立加密。块密码使用一个密钥和一个加密算法，将明文块映射为密文块。解密时使用相同的密钥和解密算法将密文块还原为明文块。块密码的安全性取决于密钥的长度和算法的复杂度。常用的块密码工作模式有 ECB（电子密码本）、CBC（密码块链接）等。

### 3.4. DES、3DES 和 AES

DES（数据加密标准）是 1977 年由美国国家标准局发布的块密码算法，使用 56 位密钥和 64 位块大小。随着计算能力的提升，56 位密钥已不够安全。3DES 使用三次 DES 加密来增强安全性，有效密钥长度为 168 位，但速度较慢。AES（高级加密标准）是 2001 年取代 DES的新标准，支持 128、192 和 256 位密钥，块大小为 128 位。AES 的安全性高、速度快，是目前最广泛使用的对称加密算法。

## 4. 公开密钥加密

### 4.1. RSA

RSA 是最著名的公开密钥加密算法，由 Rivest、Shamir 和 Adleman 在 1977 年发明。RSA 的核心思想是利用大数分解的困难性——将两个大素数相乘很容易，但将乘积分解回原来的两个素数却极其困难。每个用户拥有一对密钥：公钥公开给所有人，私钥自己秘密保存。用公钥加密的数据只有对应的私钥才能解密，反之亦然。

### 4.2. RSA 的工作原理

RSA 的密钥生成过程如下：选择两个大素数 $p$ 和 $q$，计算 $n = pq$；计算 $\phi(n) = (p-1)(q-1)$；选择一个与 $\phi(n)$ 互素的整数 $e$ 作为公钥指数；计算 $d$ 使得 $ed \equiv 1 \pmod{\phi(n)}$ 作为私钥指数。公钥是 $(n, e)$，私钥是 $(n, d)$。加密时计算 $c = m^e \mod n$，解密时计算 $m = c^d \mod n$。安全性基于大数分解问题的计算困难性。

### 4.3. 会话密钥

由于 RSA 的加密速度远慢于对称加密（约慢 100 到 1000 倍），实际应用中通常将两者结合使用。通信双方先用 RSA 交换一个随机生成的会话密钥（对称密钥），然后用这个会话密钥和 AES 等对称算法来加密实际的通信数据。这种方式结合了公钥加密的密钥分发便利性和对称加密的高效性，是 TLS 等安全协议的基本模式。
