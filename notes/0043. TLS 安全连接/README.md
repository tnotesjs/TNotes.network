## 3. TLS 的宏观描述

### 3.1. 握手阶段

TLS 握手是客户端和服务器在开始传输数据之前建立安全连接的过程。握手阶段完成三件事：协商加密算法和协议版本、验证服务器身份（通过证书）、建立共享的会话密钥。握手过程需要两个往返（2-RTT），但 TLS 1.3 将其优化为一个往返（1-RTT），甚至支持 0-RTT 恢复之前的会话。

### 3.2. 密钥导出

握手阶段完成后，客户端和服务器各自从协商的密钥材料中导出会话密钥。会话密钥包括用于加密数据的对称密钥和用于计算 MAC 的密钥。密钥导出使用伪随机函数（PRF），确保即使密钥材料被部分泄露，攻击者也无法推导出会话密钥。

### 3.3. 数据传输阶段

会话密钥建立后，客户端和服务器之间的所有应用数据都经过加密和完整性保护后传输。TLS 记录协议将数据分割成记录，每个记录独立加密和认证。接收方收到记录后先验证 MAC 确保完整性，然后解密获取原始数据。

## 4. 更完整的描述

### 4.1. TLS 握手协议

TLS 握手协议的详细过程如下。客户端发送 ClientHello 消息，包含支持的 TLS 版本、密码套件列表和一个随机数。服务器回复 ServerHello 消息，选定的密码套件和另一个随机数，同时发送自己的证书。客户端验证证书的合法性，提取服务器的公钥。客户端生成一个预主密钥（pre-master secret），用服务器的公钥加密后发送给服务器。双方使用两个随机数和预主密钥计算出主密钥，进而导出会话密钥。

### 4.2. TLS 记录协议

TLS 记录协议负责将应用数据封装成安全的记录。每个记录包含内容类型、版本号、长度、加密的数据和 MAC。发送方将应用数据分块，为每块计算 MAC，然后加密（先 MAC 后加密）。接收方先解密，再验证 MAC。TLS 1.3 使用 AEAD（认证加密与关联数据）加密模式，将加密和认证合二为一，更加高效安全。

### 4.3. 密码套件

密码套件定义了 TLS 会话中使用的一组加密算法，包括密钥交换算法、对称加密算法和散列函数。例如 `TLS_RSA_WITH_AES_128_CBC_SHA256` 表示使用 RSA 进行密钥交换、AES-128-CBC 进行加密、SHA-256 进行完整性校验。TLS 1.3 大幅精简了密码套件，移除了不安全的算法（如 RSA 密钥交换、CBC 模式），只保留了基于 ECDHE 的密钥交换和 AEAD 加密模式。

## 5. 防火墙与入侵检测

### 5.1. 传统分组过滤器

防火墙是最基本的网络安全设备，部署在网络边界处，控制进出网络的流量。传统分组过滤器根据数据包的首部信息（源/目的 IP 地址、源/目的端口号、协议类型）来决定允许还是拒绝数据包通过。管理员可以配置访问控制列表（ACL），指定哪些流量可以通过、哪些应该被阻断。

### 5.2. 状态分组过滤器

状态分组过滤器在传统过滤器的基础上增加了连接状态的跟踪。它维护一个连接状态表，记录每个活动连接的信息。只有属于已建立连接的数据包才被允许通过，新的连接需要通过额外的检查。这比传统过滤器更安全，因为它能防止攻击者利用已建立连接的漏洞注入恶意数据包。

### 5.3. 应用网关

应用网关（也叫代理防火墙）工作在应用层，能够深入检查数据包的内容。它不仅检查首部信息，还能检查应用层的载荷，识别和过滤特定的应用层内容。例如应用网关可以检查 HTTP 请求中是否包含恶意代码，或者阻止特定类型的文件下载。应用网关的安全性最高，但处理开销也最大。

### 5.4. 入侵检测系统

入侵检测系统（IDS）监控网络流量或系统日志，检测可疑的活动。基于特征的 IDS 使用已知攻击的特征库来匹配流量中的模式，类似于杀毒软件的工作方式。基于异常的 IDS 建立正常行为的基线，当检测到偏离基线的异常行为时发出警报。IDS 通常是被动的（只检测不阻断），而入侵防御系统（IPS）则能主动阻断检测到的攻击。
