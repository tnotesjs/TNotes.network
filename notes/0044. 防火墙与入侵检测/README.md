## 3. 防火墙

### 3.1. 传统分组过滤器

传统分组过滤器是最基本的防火墙类型，部署在网络的边界路由器上。它根据数据包的首部字段来做出过滤决策：源和目的 IP 地址、源和目的端口号、协议类型（TCP、UDP、ICMP 等）。管理员通过配置访问控制列表（ACL）来指定允许和拒绝的规则。ACL 的规则按顺序匹配，第一条匹配的规则生效。例如一条规则可以规定「允许来自 192.168.1.0/24 子网的 HTTP 流量，拒绝其他所有流量」。

### 3.2. 状态分组过滤器

状态分组过滤器比传统过滤器更智能。它不仅检查单个数据包，还跟踪连接的状态。当看到一个 TCP SYN 包时，它知道这是一个新连接的开始，会记录到连接状态表中。只有属于已建立连接（状态表中有记录）的数据包才被允许通过。这有效防止了 SYN 洪泛攻击——攻击者发送大量伪造的 SYN 包试图耗尽服务器资源，状态防火墙可以在到达服务器之前将它们拦截。

### 3.3. 应用网关

应用网关（代理防火墙）工作在应用层，能够深度检查数据包的内容。所有进出网络的流量都必须经过应用网关的代理——客户端先与网关建立连接，网关再与服务器建立连接，两边的数据交换都经过网关的检查和过滤。应用网关可以识别应用层的攻击（如 SQL 注入、跨站脚本），但处理开销大，可能成为性能瓶颈。

### 3.4. 防火墙的局限性

防火墙虽然重要，但有明显的局限性。防火墙只能检查经过它的流量，无法防御来自网络内部的攻击。加密流量（如 HTTPS）使防火墙无法检查载荷内容，攻击者可以利用加密通道绕过防火墙的检测。防火墙的规则配置复杂，配置错误可能导致安全漏洞或正常服务被阻断。此外，防火墙无法防御零日攻击（利用未知漏洞的攻击），因为没有对应的过滤规则。

## 4. 入侵检测系统

### 4.1. 基于特征的 IDS

基于特征的 IDS 维护一个已知攻击的特征数据库。它检查网络流量中的每一个数据包，将其与特征库中的模式进行匹配。如果匹配成功，就发出警报。这种方式对已知攻击的检测率很高，但对未知攻击无能为力。特征库需要不断更新以应对新出现的攻击。基于特征的 IDS 类似于杀毒软件的工作原理。

### 4.2. 基于异常的 IDS

基于异常的 IDS 首先学习网络的正常行为模式，建立一个基线。之后持续监控网络流量，当检测到明显偏离基线的异常行为时发出警报。例如正常情况下某个服务器在工作时间每分钟处理 100 个请求，如果突然增加到 10000 个，IDS 就会认为这可能是 DDoS 攻击。基于异常的 IDS 能够检测未知攻击，但误报率较高，因为正常的流量变化也可能被误判为异常。

### 4.3. IDS 的部署

IDS 可以部署在网络的关键位置。网络 IDS（NIDS）部署在网络的入口点或核心交换机上，监控经过的流量。主机 IDS（HIDS）运行在单台主机上，监控系统日志、文件完整性和进程行为。入侵防御系统（IPS）与 IDS 类似，但能主动阻断检测到的攻击（如丢弃恶意数据包、重置连接）。实际部署中通常将 IDS/IPS 与防火墙配合使用，形成多层防御体系。
